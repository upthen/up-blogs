## 📂 数据来源

### chinese-gushiwen 数据库
- **位置**: `.material/chinese-gushiwen/guwen/guwen0-1000.json`
- **总数据量**: 10,000 首古诗文
- **提取数量**: 前 300 首
- **文件格式**: 多个 JSON 对象用逗号分隔（NDJSON 变体）

### 数据结构
```json
{
  "_id": {"$oid": "..."},
  "title": "将进酒",
  "dynasty": "唐代",
  "writer": "李白",
  "type": ["乐府", "唐诗三百首", "咏物"],
  "content": "君不见，黄河之水天上来...",
  "remark": "将进酒：属乐府旧题...",
  "translation": "你难道看不见那黄河之水...",
  "shangxi": "这首诗非常形象的表现了李白...",
  "audioUrl": "https://..."
}
```

## 🔧 技术实现

### 1. 数据提取脚本
**文件**: `scripts/extract-poems.cjs`

功能：
- ✅ 读取 `guwen0-1000.json` 文件
- ✅ 解析 NDJSON 格式（多个 JSON 对象用逗号分隔）
- ✅ 提取前 300 首诗词
- ✅ 转换为目标格式（`data/poems.ts`）
- ✅ 生成 TypeScript 接口和辅助函数

### 2. 格式转换
**原始格式** → **目标格式**:

| 原始字段 | 目标字段 | 转换逻辑 |
|----------|----------|----------|
| `_id` | `id` | 生成简化 ID（标题 + 随机字符） |
| `title` | `title` | 直接映射 |
| `writer` | `author` | 字段重命名 |
| `dynasty` | `dynasty` | 直接映射 |
| `type` | `type` | 提取主类型标签 |
| `content` | `content` | 直接映射 |
| `translation` | `notes` | 优先使用翻译，否则用注释 |

### 3. 类型映射
```typescript
const typeMap = {
  '乐府': '乐府',
  '唐诗三百首': '诗',
  '宋词': '词',
  '元曲': '曲',
  '赋': '赋'
};
```

## 📌 注意事项

### 1. 类型说明
- **诗**: 295 首，主要为唐诗
- **乐府**: 5 首，主要为李白的作品
- 暂未包含：词、曲、赋（可以在后续扩展）

### 2. ID 生成规则
```typescript
id = title
  .replace(/[^a-zA-Z0-9\u4e00-\u9fa5]/g, '')
  .substring(0, 20)
  + '-' + Math.random().toString(36).substring(2, 6);
```

### 3. Notes 字段
优先级：
1. `translation`（翻译）
2. `remark`（注释）
3. 空字符串

### 4. 兜底策略
- API 失败时随机选择 300 首之一
- 不保存到 localStorage（下次仍尝试 API）
- 保证用户始终能看到诗词
